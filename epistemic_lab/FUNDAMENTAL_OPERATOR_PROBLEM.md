# Research Program IV: The Fundamental Operator Problem

Status: theoretical program only.

This document suspends the search for new macrovariables. The previous program
proposed:

```text
B0 = {G, R, K, C, M}
```

where `G` is generative pressure, `R` retention pressure, `K` coupling strength,
`C` compression load and `M` mobility / transformability.

The new hypothesis is more severe:

```text
B0 may not be a fundamental basis.
```

It may be a five-coordinate shadow of a deeper epistemic dynamic. The goal is
to discover whether one minimal dynamic object can generate distinctions,
concepts, invariants, composition, hierarchy, generalization, transfer and
relevance criteria.

No code. No new objectives. No new algorithms.

## 1. Do Not Start With an Operator

The symbol `Phi` is not allowed as a starting assumption.

We must not assume that the fundamental object is a function, operator,
category, functor, monoid, lattice, order, flow, relation or any other known
mathematical type. The first question is:

```text
What is the minimal object that changes during knowing?
```

Only after deriving the required properties may we ask whether the resulting
object resembles an existing mathematical structure.

## 2. Candidate Things That Might Change

Several objects can change during epistemic growth:

1. A set of candidate models.
2. A set of admissible explanations.
3. A relation of distinguishability.
4. A family of quotients.
5. A constraint system.
6. A partial ordering of adequacy, refinement or consequence.
7. A space of possible future tests.
8. A rule assigning relevance to distinctions.
9. A coupling between current structure and future interaction.

None is obviously fundamental.

### 2.1 Candidate: Set of Models

A changing set of models can express birth and death of hypotheses. But it is
too weak by itself.

Counterexample:

```text
M_t = {m1, m2, m3}
M_{t+1} = {m1, m2, m4}
```

The set changed, but the expression says nothing about which model explains
which distinction, which future tests are preserved, or whether `m4` is a new
concept, a duplicate, a relabeling or noise.

Therefore "set of models" is not enough.

### 2.2 Candidate: Set of Explanations

Explanations are closer because they connect models to constraints. But a bare
set of explanations still lacks comparison.

Open knowledge growth requires knowing when one explanation replaces, refines,
generalizes or composes with another. Membership alone cannot express this.

Therefore a set of explanations is not enough.

### 2.3 Candidate: Distinguishability Relation

A relation of distinguishability is central to previous theory:

```text
x and y are distinguishable by admissible future tests
```

This captures the birth and removal of distinctions. But it still does not
capture all epistemic growth.

Counterexample:

Two systems may have the same distinguishability relation but different
compositional resources. One can combine two stable distinctions into a reusable
theory; the other stores them as unrelated facts. Their distinguishability
relations match, but their open-ended growth differs.

Therefore distinguishability is necessary but probably not sufficient.

### 2.4 Candidate: Family of Quotients

A family of quotients is richer than one quotient. It can express multiple
levels of abstraction and multiple test families.

But quotient families are still retrospective unless they include how future
tests act on them. A quotient can be sufficient for yesterday's tests and fail
tomorrow's interaction.

Therefore quotient families are not enough unless they are coupled to future
admissibility.

### 2.5 Candidate: Constraint Space

A constraint space may be more fundamental:

```text
knowledge = restrictions on what can still be the case
```

This handles models, explanations and tests as derived objects. A model is a
way to satisfy constraints. A distinction is forced when two possibilities no
longer satisfy the same constraints. A quotient is a compression of constraints.

But constraints alone do not provide growth. A system can accumulate arbitrary
constraints with no relevance, compression or transfer.

Therefore the changing object cannot be just a constraint set. It needs a notion
of consequence under future interaction.

## 3. Minimal Thing That Must Change

The minimal object appears to be neither a model set nor a quotient.

It is a changing structure of admissible consequence:

```text
At a time t, the system carries a way of saying which distinctions,
explanations or models have the same consequences for future admissible tests.
```

Call this object, temporarily:

```text
E_t = epistemic consequence structure at time t
```

This name deliberately avoids choosing a mathematical type.

`E_t` must support at least four questions:

1. Which alternatives are still distinguishable?
2. Which distinctions matter for admissible future interaction?
3. Which structures can replace, refine or compose with which others?
4. Which changes preserve relevant consequences?

If an object cannot answer these questions, it cannot support open knowledge
growth.

## 4. Minimal Operations Must Be Derived

Do not assume merge and split.

A system capable of open knowledge accumulation must support transformations
with the following roles.

### 4.1 Differentiation

The system must be able to make a previously unavailable consequential
difference available.

This need not be a split of an equivalence class. It may be:

```text
new constraint
new test consequence
new latent variable
new explanatory dependency
new failure of a former substitution
```

Observable shadow:

```text
births, depth growth, new branches, rising entropy over candidates
```

This role later projects to `G`.

### 4.2 Stabilization

The system must be able to preserve a consequential structure through further
interaction.

This need not be persistence of the same entity. A structure may persist by
being re-expressed in a new representation.

Observable shadow:

```text
lifetime, survival, valid reuse, low destruction rate
```

This role later projects to `R`.

### 4.3 Substitution

The system must determine when one structure can stand in for another without
changing admissible consequences.

This is more primitive than compression. Compression is what substitution looks
like when the replacement is smaller.

Observable shadow:

```text
compression ratio, quotient formation, semantic subsumption
```

This role later projects to `C`.

### 4.4 Composition

The system must be able to combine retained structures so that the combined
structure has consequences not reducible to isolated storage.

This is not just graph branching. Branching is one coordinate expression of
composition.

Observable shadow:

```text
reuse, hierarchy, depth, theory-like structures
```

This role later projects partly to `K`.

### 4.5 Transport

The system must be able to say when a structure remains the same under a change
of representation, objective, world interface, feature basis or agent
implementation.

Without transport, there is no invariant macrovariable. There are only
coordinate-dependent measurements.

Observable shadow:

```text
transfer, feature-ablation stability, objective/world invariance
```

This role later projects to `M`.

### 4.6 Relevance Filtering

The system must reject or ignore differences that do not matter for admissible
future consequences.

This is not optional. Without relevance filtering, open growth degenerates into
unbounded accumulation of arbitrary differences.

Observable shadow:

```text
death, pruning, non-survival, stable low-dimensional regimes
```

This role is distributed across `R`, `C` and `M`.

## 5. First Derived Object: Consequence-Transforming Dynamics

The minimal dynamic object is not yet a function.

It is a rule, process or constraint on transitions:

```text
E_t  ->  E_{t+1}
```

where `E_t` is an epistemic consequence structure. This transition must preserve
enough identity to make persistence meaningful, but it must allow enough change
to support new distinctions and new relevance criteria.

If this transition can be represented as an operator, we may later write:

```text
Phi(E_t) = E_{t+1}
```

But that notation is optional. The fundamental claim is weaker:

```text
There exists a dynamic law on consequence structures.
```

The object is therefore:

```text
D = (admissible consequence structures, allowed transitions between them)
```

This is the current best candidate. It is not just a state space and not just a
map. It is a space of possible epistemic consequence structures together with
the admissible ways they can change.

## 6. Deriving B0 From the Dynamic Object

If `D` is correct, then `B0` is not fundamental. Its elements are observable
aspects of transitions in `D`.

### 6.1 G: Differentiation Rate

`G` is the degree to which a transition forces consequential distinctions that
were not available in the previous structure.

```text
G = observable rate of necessary differentiation in D
```

It is high when future admissible consequences require new separations,
dependencies or explanatory forms.

### 6.2 R: Stabilization Strength

`R` is the degree to which consequential structure persists across allowed
transitions.

```text
R = observable persistence of consequence under D
```

It is high when a distinction, explanation or quotient remains consequence
preserving over many transitions, even if its representation changes.

### 6.3 K: Compositional Coupling

`K` is the degree to which structures participate in the future consequences of
other structures.

```text
K = observable coupling among retained consequence structures
```

It is high when one structure is repeatedly usable in different contexts because
it is connected to the consequence structure of many other objects.

### 6.4 C: Substitution Pressure

`C` is the degree to which the dynamics admits consequence-preserving
replacement by a smaller or more general structure.

```text
C = observable pressure toward sufficient substitution
```

It is not primitive compression. It is the compression-like face of
substitution under a resource or representation boundary.

### 6.5 M: Transport Coherence

`M` is the degree to which consequence structures and their transitions commute
with changes of representation, world, objective, feature basis or agent
implementation.

```text
M = observable coherence of D under transformation of coordinates
```

It is high when the same epistemic structure can be transported across
coordinate systems without losing its future consequences.

## 7. Are Any B0 Variables Independent?

The reduction is not complete.

`G`, `R`, `K`, `C` and `M` are derivable only if `D` includes:

1. consequential distinguishability;
2. allowed transitions;
3. substitution;
4. composition;
5. transport across coordinate changes.

If transport is absent, `M` is independent.

If composition is absent, `K` collapses to recurrence and loses theory-like
meaning.

If substitution is absent, `C` cannot be derived.

If only static consequence is present and no transition is present, neither `G`
nor `R` can be derived.

Therefore the one-object hypothesis survives only for a dynamic consequence
structure rich enough to include these roles. A bare operator on model states is
too weak.

## 8. Fixed Structures

Now fixed points can be discussed without assuming ordinary functions.

A fixed structure is not necessarily `Phi(x) = x`. More generally:

```text
An epistemic structure is fixed if all allowed future transitions preserve its
admissible consequences up to permitted transport.
```

This gives a representation-independent notion:

```text
fixed = consequence-stable under D
```

### 8.1 Are Concepts Fixed Structures?

Possibly, but only under a restricted interpretation.

A concept is not a fixed block of states. A concept would be a consequence
structure that remains substitutable and transportable across admissible future
tests.

So:

```text
concept ~= fixed consequential role
```

not:

```text
concept = stable label
```

### 8.2 Are Abstractions Stable Manifolds?

This has mathematical sense if "nearby" structures exist. But nearness is not
fundamental unless derived from consequence similarity or substitutability.

Tentative interpretation:

```text
abstraction = region of consequence structures whose transitions preserve a
shared quotient role
```

Calling this a manifold is premature. It may be only a basin, preorder region,
equivalence class, filter or other object.

### 8.3 Are Theories Compositions of Fixed Structures?

This has sense if composition is part of `D`.

A theory would not be a set of concepts. It would be a composite consequence
structure whose components are individually stable and whose composition is also
stable under future admissible tests.

Tentative form:

```text
theory = stable composition of fixed consequential roles
```

### 8.4 Are Scientific Revolutions Basin Transitions?

This has sense if `D` contains regions of attraction.

A revolution would not be ordinary belief update. It would be a transition in
which the old fixed structures stop being transportable or substitutable, and a
different family of structures becomes stable.

Tentative form:

```text
scientific revolution = change of attracting consequence structure
```

This is meaningful only after attraction is defined without assuming a metric or
topology.

## 9. Minimal Order Needed for Fixed Structures

Do fixed structures require an order?

Some comparison relation seems necessary, but it need not be a full order.

The weakest needed comparison appears to be:

```text
a can substitute for b with respect to admissible consequences
```

Call it:

```text
a <= b
```

with the intended reading:

```text
a is no less consequence-sufficient than b
```

But this notation is only a placeholder.

### 9.1 Required Properties

Reflexivity seems necessary:

```text
a can substitute for itself
```

Without reflexivity, persistence cannot even be stated.

Transitivity may be necessary:

```text
if a substitutes for b, and b substitutes for c, then a substitutes for c
```

But transitivity can fail when substitution is context-dependent. Open
knowledge growth can still occur with local substitution rules, so global
transitivity is not fundamental unless contexts are included in the objects.

Antisymmetry is not necessary. Two different representations may substitute for
each other without being identical.

Completeness is not necessary. Many structures may be incomparable.

Lattice joins/meets are not necessary. Composition may exist only partially.

Continuity is not necessary at this level. It may become useful only after a
notion of approximation is derived.

### 9.2 Minimal Comparison Structure

The current minimal requirement is:

```text
context-indexed substitutability relation
```

not a lattice, not a complete order and not necessarily a global partial order.

Fixed structures require only that we can state:

```text
after transition, the transported successor can substitute for the predecessor
with respect to the relevant future contexts
```

This is weaker than standard fixed-point theory.

## 10. Destruction Tests

Every assumption must be attacked.

### 10.1 Does Open Knowledge Need Models?

No.

A system may accumulate constraints, rules, skills or predictive equivalence
classes without explicit models. Models are not fundamental.

### 10.2 Does Open Knowledge Need Quotients?

No as primitive objects.

Quotients arise whenever consequence-preserving identification is possible, but
a system can represent knowledge as programs, constraints or transformations
instead of explicit quotient sets.

### 10.3 Does Open Knowledge Need Merge/Split?

No.

Merge/split are coordinate expressions of substitution and differentiation.
They are not fundamental operations.

### 10.4 Does Open Knowledge Need a Global Order?

No.

Contextual substitutability can be non-total and locally non-transitive if the
context is omitted. A global order is therefore not fundamental.

### 10.5 Does Open Knowledge Need Fixed Points?

No for short-term adaptation.

Yes for stable concepts or theories. Open accumulation can proceed without
settled fixed structures, but cumulative knowledge requires some structures to
remain consequence-stable long enough to compose with later structures.

### 10.6 Does Open Knowledge Need Transport?

For ordinary learning in one closed representation, no.

For representation-invariant epistemology, yes. Since this program requires
invariance across objective, representation, world, features and implementation,
transport is fundamental to this project.

### 10.7 Does Open Knowledge Need Compression?

No as size reduction.

Yes as substitution. Without some consequence-preserving replacement relation,
the system cannot distinguish knowledge growth from unlimited storage.

### 10.8 Does Open Knowledge Need Composition?

For finite memorization, no.

For open-ended growth, yes. Without composition, new knowledge cannot become
material for later knowledge except as isolated entries.

## 11. Comparison With Existing Theories

This comparison comes only after the internal derivation.

### 11.1 MDL

MDL captures the substitution/compression face of `D`. It does not by itself
derive admissible consequence, transport or composition. It is too narrow to be
the full object.

### 11.2 Information Bottleneck

Information Bottleneck captures relevance-preserving compression relative to a
chosen variable. It depends on the target variable. The fundamental object here
must explain where relevance targets come from and how they change.

### 11.3 Causal States

Causal states are close to fixed consequence roles for prediction: pasts are
equivalent when they have the same future distribution. They capture one strong
instance of consequence structure, but mostly for predictive futures rather than
arbitrary admissible tests.

### 11.4 Automaton Minimization

Automaton minimization is a quotient construction over future distinguishability.
It captures a precise special case of stable consequence equivalence. It does
not include changing test families, transport or open-ended composition.

### 11.5 Category Theory

Category theory may describe objects, transformations, composition and
transport. But adopting it at the start would be premature. If the derived
structure requires objects plus composable transformations plus equivalence up
to transport, categorical language may become natural later.

### 11.6 Domain Theory

Domain theory may become relevant if knowledge states form an approximation
order. The current derivation does not justify completeness, directed joins or
continuity. Those would be extra axioms.

### 11.7 Abstract Interpretation

Abstract interpretation studies sound abstraction relative to properties. It is
close to consequence-preserving substitution. But it normally begins with a
semantic domain and correctness relation already specified. This program asks
where those come from.

### 11.8 Statistical Mechanics

Statistical mechanics is analogous in its separation of microstate coordinates
from macrovariables. The analogy is useful only if `G`, `R`, `K`, `C` and `M`
become state variables derived from the deeper dynamics, not imported names.

### 11.9 Renormalization Group

Renormalization is close to repeated consequence-preserving transformation under
scale change. But the epistemic object need not be scale-based. The safer
analogy is transformation-stable structure, not literal renormalization.

### 11.10 Fixed-Point Theory

Fixed-point theory becomes relevant only after the dynamic object has enough
structure to define preservation under transition. Classical fixed points may
be too strict; transported fixed structures are more plausible.

### 11.11 Dynamical Systems

Dynamical systems language becomes useful if the consequence structures form a
state space with trajectories and attraction. The current derivation does not
yet justify topology, metric, smoothness or continuity.

## 12. Current Verdict

The fundamental object is not a bare operator on models.

The best current candidate is:

```text
D = a dynamic structure of admissible consequences, equipped with
    differentiation, stabilization, substitution, composition and transport
    roles.
```

If representable as an operator, it may later be written as:

```text
Phi: E_t -> E_{t+1}
```

But `Phi` is notation, not ontology.

The deeper claim is:

```text
open knowledge growth requires dynamic consequence structure.
```

`B0 = {G, R, K, C, M}` is then a set of macroscopic observables:

```text
G = differentiation
R = stabilization
K = compositional coupling
C = substitution pressure
M = transport coherence
```

## 13. Next Theoretical Tasks

1. Define "admissible consequence" without assuming a test family fixed in
   advance.
2. Define context-indexed substitutability precisely.
3. Determine whether composition is primitive or derivable from
   substitutability plus consequence.
4. Determine whether transport is primitive or a higher-order form of
   substitutability.
5. Define fixed consequential roles without assuming equality of states.
6. Search for a counterexample where all five roles exist separately but cannot
   be unified into one dynamic consequence structure.
7. Search for a counterexample where open knowledge growth exists without one
   of the five roles.

Success means either:

```text
one minimal dynamic consequence object derives B0 and the observed epistemic
phenomena
```

or:

```text
the failure identifies a smaller irreducible axiom set that cannot be reduced
without losing open-ended knowledge growth
```
