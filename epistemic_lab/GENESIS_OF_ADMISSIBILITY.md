# Phase IV: Genesis of Admissibility

Status: new theoretical program.

The Reduction Program is closed. We no longer search for weaker primitives
below `K`.

The new question is:

```text
How can K arise if no admissibility boundary is given at the start?
```

Do not start with `K`.

Do not assume admissibility, inadmissibility, world, models, consequence,
causality, time or tests. If any of these becomes necessary, it must be derived
or explicitly introduced as an axiom.

No code. No algorithms. No new objectives.

## 1. Starting Point: Raw Interactions

Begin with a completely unstructured collection:

```text
I = raw interactions
```

At this level, an interaction is only an occurrence-token.

No assumptions:

1. no order;
2. no time;
3. no repetition;
4. no identity across occurrences;
5. no types;
6. no success or failure;
7. no compatibility;
8. no contradiction;
9. no agent;
10. no world;
11. no future;
12. no tests.

This is deliberately too weak. The first task is to prove how weak it is.

## 2. Negative Theorem 1: No Boundary From a Bare Set

### Claim

A bare set of interaction tokens cannot generate a nontrivial admissibility
boundary non-arbitrarily.

### Argument

Let:

```text
I = {i1, i2, ..., in}
```

with no relations, labels, order, repetitions, operations or observations.

Any proposed boundary:

```text
K subset Fin(I)
```

selects some finite collections as admitted and others as non-admitted.

But because `I` has no structure, every permutation of `I` is an automorphism.
There is no internal reason to select one token, pair or finite subset rather
than another of the same cardinality.

Therefore any nontrivial `K` must come from outside the bare set.

### Stronger Form

If a construction rule for `K` is invariant under all permutations of a bare
set, then `K` can depend only on cardinality:

```text
X in K iff |X| belongs to A
```

for some set of allowed sizes `A`.

But choosing `A` is itself an external admissibility criterion unless it is
derived from additional structure.

Thus even cardinality-based `K` is not generated by interaction content. It is
injected by the rule.

### Verdict

From a bare set alone:

```text
non-arbitrary K is impossible.
```

## 3. Negative Theorem 2: No K Without Difference

### Claim

If all raw interactions are indistinguishable under every available internal
transformation, then no nontrivial admissibility boundary can arise.

### Argument

Suppose every interaction token has the same structural role. Then any
candidate distinction between configurations must classify structurally
identical objects differently.

That classification is not induced. It is labeling.

Therefore:

```text
difference is necessary for K.
```

But not every difference is enough. Arbitrary labels create arbitrary `K`.
The required difference must be constrained by something not chosen after the
desired boundary.

## 4. Negative Theorem 3: No Stable K Without Recurrence

### Claim

A stable admissibility boundary cannot arise from one-shot unrelated
interactions.

### Argument

If each interaction occurs once and has no relation to any other interaction,
then there is no basis for saying that a boundary persists across cases.

One can define:

```text
this token admitted
that token not admitted
```

but this is a token-level mark, not a stable boundary. It does not generalize to
new configurations.

Therefore stable `K` requires some form of recurrence:

```text
not necessarily time,
but repeated structural availability of comparable situations.
```

### Important Limitation

Recurrence is not yet time. It is weaker:

```text
there exist multiple occurrences that can be compared as instances of a
common pattern or role.
```

Time may later organize recurrence, but recurrence does not require time.

## 5. Negative Theorem 4: No Non-Injected K Without Invariance

### Claim

If the proposed boundary is not invariant under admissible relabelings of raw
interaction tokens, it is injected.

### Argument

Before `K`, token names have no epistemic meaning. A boundary that depends on
token names rather than structural relations cannot be discovered from
interactions.

Therefore any non-injected `K` must satisfy:

```text
if two configurations have the same generated structure,
then they receive the same admissibility status.
```

This is the first appearance of invariance.

Invariance is not optional. It is what separates generated admissibility from
post-hoc labeling.

## 6. Minimal Impossibility Conditions

Emergence of nontrivial, non-injected `K` is impossible if the starting material
has all of the following:

1. no distinguishable differences;
2. no recurrence or comparability;
3. no relation among interaction tokens;
4. no invariant construction rule;
5. no transformation under which arbitrary labels can be ignored.

This is the weakest current impossibility result:

```text
pure unstructured multiplicity cannot generate admissibility.
```

Some additional principle is necessary.

## 7. What Minimal Addition First Makes K Possible?

The minimal addition cannot be `K` itself.

It also should not be consequence, tests, causality or time.

The weakest candidate is:

```text
co-variation / co-occurrence structure over raw interactions
```

Not admissibility. Not compatibility. Only a relation saying that some
interaction tokens occur together, alternate with one another, or can be
compared as part of the same presentation.

Represent this generically as:

```text
R over finite subsets of I
```

Read:

```text
R(X) means X is presented together as a raw configuration
```

This is not yet `K`. `R(X)` does not mean `X` is admissible. It means only that
`X` is part of the raw interaction data.

## 8. Why R Is Weaker Than K

`R` is descriptive:

```text
this configuration is encountered / presented / co-occurs
```

`K` is normative or structural:

```text
this configuration is admissibly realizable
```

From one presentation, no admissibility follows. But from recurrent
presentations and absences, a boundary may become identifiable.

The key move:

```text
K may arise as the invariant closure of recurrent presentation structure.
```

## 9. Minimal Genesis Principle

### Principle GP1: Invariant Recurrence

If raw interactions generate repeated comparable presentations, then any
candidate admissibility boundary must be invariant under the transformations
that preserve the recurrence structure.

Form:

```text
Given (I, R), derive K only from features invariant under automorphisms of
(I, R).
```

This prevents token-label injection.

But GP1 alone may produce only the observed presentation set:

```text
K = {X : R(X)}
```

That is too narrow if non-observed but structurally forced configurations
should count as admissible.

### Principle GP2: Conservative Closure

The first admissibility boundary is the weakest closure of recurrent
presentations that does not introduce distinctions unsupported by recurrence.

This is intentionally abstract. It does not specify an algorithm. It states a
minimality requirement:

```text
include what recurrence forces;
exclude what recurrence rules out;
leave undecided what recurrence does not constrain.
```

This suggests that the earliest object may be not a two-valued `K`, but a
three-zone boundary:

```text
K+ = forced admissible
K- = forced inadmissible
K? = undetermined
```

A sharp `K` appears only when enough structure collapses the undecided zone.

## 10. Negative Theorem 5: Sharp K Does Not Always Emerge

### Claim

Even with recurrence, a unique sharp `K` need not arise.

### Argument

Suppose the raw recurrence structure supports two invariant closures:

```text
K1
K2
```

both compatible with all observed presentations and all observed absences, but
differing on unobserved configurations.

Then the data do not determine a unique `K`.

Any choice between `K1` and `K2` requires an additional principle:

1. minimal closure;
2. maximal closure;
3. simplicity;
4. symmetry;
5. stability under later interaction;
6. external intervention.

Therefore:

```text
emergent K is generally underdetermined.
```

## 11. Multiple K

Multiple admissibility boundaries are not a bug. They are expected.

Different `K` can arise from the same raw interaction collection if they
preserve different invariant structures:

```text
K_structural
K_operational
K_communicative
K_memory
K_compositional
```

But these names are interpretations. Formally, they are different invariant
closures over the same recurrence presentation.

This means the previous theory should not assume a single global `K` unless a
unification principle is proven.

## 12. Can K Change?

Yes, if the recurrence structure changes or if the transformation group used to
ignore arbitrary labels changes.

Without assuming time, define change as a path through presentations:

```text
(I_alpha, R_alpha) indexed by stages alpha
```

The index `alpha` need not be physical time. It may be refinement, expansion of
available interactions, or change of presentation scope.

Then:

```text
K_alpha = invariant closure of (I_alpha, R_alpha)
```

So dynamic `K` is derived from changing recurrence structure.

## 13. Stabilization of K

`K` stabilizes when further presentations do not change the induced invariant
closure.

Form:

```text
K_alpha stabilizes at beta iff for all gamma >= beta,
the induced K_gamma is equivalent to K_beta under admissible relabeling.
```

This uses an index order over stages. If no stage comparison exists, stability
cannot be stated.

Therefore a theory of stabilization requires an additional weak structure:

```text
comparability of presentation stages.
```

This is weaker than time but stronger than a bare set.

## 14. Source of Stability

The source of stability is not assumed to be causality, environment or tests.

The minimal source is:

```text
persistent recurrence invariants.
```

Examples of possible realizations:

1. repeated co-occurrence;
2. repeated exclusion;
3. repeated role alternation;
4. repeated failure of co-presentation;
5. closure under transformations;
6. independent recovery of the same boundary from different presentations.

These can later be interpreted as environmental constraints, compatibility,
contradiction minimization or interaction regularities. But at this phase they
are only recurrence invariants.

## 15. Deriving the Previous Theory

Once an emergent `K` exists, the previous theorem inventory applies.

From:

```text
K_alpha
```

derive:

```text
profile_alpha(x) = {C : C union {x} in K_alpha}
```

Then:

```text
equivalence:
  x ~= y iff profile_alpha(x) = profile_alpha(y)

substitutability:
  x >= y iff profile_alpha(y) subset profile_alpha(x)

quotient:
  Q_alpha = U / ~=_alpha

abstraction:
  stable profile class under admissible stage changes

hierarchy:
  structured inclusion/refinement among profile classes, if such inclusion is
  nontrivial and compositional
```

Important limitation:

Abstractions and hierarchies do not follow from raw `K_alpha` alone. They
require stability and nontrivial profile inclusion across stage changes.

## 16. Current Genesis Theory

The current best structure is:

```text
raw interactions I
  -> recurrence / presentation relation R
  -> invariance under relabeling of (I, R)
  -> conservative closure
  -> emergent partial boundary (K+, K-, K?)
  -> sharp K only when undecided cases collapse
  -> profiles, equivalence, substitutability, quotient
```

This is not a learning algorithm. It is a minimal genesis schema.

## 17. Axioms Introduced and Why

### Axiom G1. Presentation Relation

There is some relation over finite raw interaction collections:

```text
R(X)
```

Reason:

A bare set cannot generate non-arbitrary `K`.

### Axiom G2. Invariance

The construction of admissibility must be invariant under automorphisms of
`(I, R)`.

Reason:

Without invariance, `K` can be injected by token names.

### Axiom G3. Conservative Closure

The induced boundary must not decide more than recurrence structure supports.

Reason:

Without conservatism, arbitrary unobserved configurations can be labeled
admissible or inadmissible.

### Optional Axiom G4. Stage Comparability

There is a way to compare presentation stages:

```text
alpha <= beta
```

Reason:

Needed only for emergence over stages and stabilization. Not needed for one
static genesis step.

## 18. Final Verdict

The emergence of `K` from a completely unstructured set is impossible.

The weakest current impossibility condition is:

```text
no relations + no recurrence + no invariance = no non-injected K
```

The minimal additional principle that first makes `K` possible is not
compatibility, consequence, causality, tests or time.

It is:

```text
invariant recurrence structure over raw interactions.
```

Even then, a unique sharp `K` is not guaranteed. The more honest first object is
a partial boundary:

```text
(K+, K-, K?)
```

A sharp `K` emerges only when recurrence and closure principles determine all
relevant configurations, or when an additional sharpening principle is
introduced and justified.

The next research question is:

```text
What sharpening principles are non-arbitrary, and can any be derived from
invariant recurrence alone?
```
