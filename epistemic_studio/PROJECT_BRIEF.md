# Claude Code

# Project: Epistemic Studio

Everything developed so far (Epistemic Engine, Lab, Explorer) is considered Version 1.0.

Do NOT extend the theory.

Do NOT invent new epistemic concepts.

Freeze the theory.

The objective is to build an operating system for research.

---

# Vision

Epistemic Studio is not another agent framework.

It is a persistent scientific laboratory where multiple autonomous researchers continuously improve a shared Research State.

The system should resemble a real scientific institute rather than a chatbot.

---

# Core Principle

Agents never communicate by chat history.

Agents communicate only through explicit scientific artifacts.

Every iteration updates the shared Research State.

---

# Research State

Design a persistent data model containing at least:

Research Question

Hypotheses

Assumptions

Experiments

Counterexamples

Observations

Applicability Atlas

Confidence

Open Questions

Contradictions

Research History

Derived Concepts

Invariant Candidates

Unknown Regions

Every object must have unique IDs, timestamps, provenance and links.

Nothing disappears.

Knowledge evolves.

---

# Agents

Implement independent long-lived agents.

Minimum set:

Explorer

* searches unexplored regions
* proposes hypotheses
* finds missing connections

Lab

* develops theories
* formalizes ideas
* derives consequences

Engine

* destroys hypotheses
* searches counterexamples
* proposes falsification experiments

Historian

* maintains Research State
* merges duplicates
* updates confidence
* builds research timeline

Cartographer

* updates the global knowledge map
* measures simplification
* tracks unresolved regions

Planner

* chooses next experiments
* maximizes epistemic value
* minimizes redundant work

Meta Observer

* evaluates the research process itself
* detects stagnation
* detects paradigm shifts
* proposes process improvements

---

# Research Cycle

Forever repeat:

Observe Research State

↓

Choose highest-value frontier

↓

Run parallel investigation

↓

Merge results

↓

Resolve contradictions

↓

Update Research State

↓

Measure simplification

↓

Choose next frontier

# Memory

Memory is the central object.

Agents are disposable.

Research State is permanent.

No agent owns knowledge.

Knowledge belongs to the Studio.

---

# Success Metric

The objective is NOT:

more hypotheses

more papers

more graphs

The objective is:

A simpler and more coherent understanding of the domain after every research cycle.

The primary KPI is:

Compression of understanding.

Measure whether many independent hypotheses become explainable by fewer, deeper concepts.

---

# Long-Term Goal

After thousands of iterations the Studio should behave less like a collection of LLM agents and more like a continuously evolving scientific civilization with persistent memory, explicit reasoning, accumulated failures, reusable concepts and self-improving research strategy.

The output of the Studio is not text.

The output of the Studio is an increasingly accurate and compressed map of reality.
