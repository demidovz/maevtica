# Necessity Classification

## persistent_memory
Classification: necessary_function
Derived from: A1
Replaceable by: biological weights, distributed traces, versioned records, cultural transmission
Lost if removed: Earlier evidence cannot constrain later belief; cumulative knowledge collapses into repeated first attempts.
Derivation: From A1, some state must persist. A literal research database is replaceable, but retention is necessary.

## institutionalized_criticism
Classification: emergent_consequence
Derived from: A2, A3
Replaceable by: adversarial testing, prediction markets, selection pressure, independent replication
Lost if removed: Errors can persist without a channel that forces alternatives to compete against evidence.
Derivation: Criticism is one social implementation of error-sensitive update over discriminable alternatives.

## preserved_failures
Classification: emergent_consequence
Derived from: A1, A2, A4
Replaceable by: negative gradients, test failures, replication failures, selection against bad variants
Lost if removed: The system repeats known bad regions or overgeneralizes beyond defeated scope.
Derivation: Failures must affect future updates, but they need not be preserved as human-readable cases.

## question_refinement
Classification: emergent_consequence
Derived from: A3, A6
Replaceable by: feature learning, model class revision, ontology update, problem decomposition
Lost if removed: The system remains trapped in a representation that may make improvement impossible or inefficient.
Derivation: When alternatives are represented badly, improvement requires changing the question or representation.

## knowledge_compression
Classification: contingent_but_expected_under_scarcity
Derived from: A5
Replaceable by: cached policies, abstractions, compiled heuristics, generalized models
Lost if removed: Without reusable structure, the system may still learn by lookup if resources are abundant, but loses efficiency under scarcity.
Derivation: Compression follows from resource-bounded reuse, not from cumulative knowledge alone.

## applicability_boundaries
Classification: necessary_function
Derived from: A4
Replaceable by: context tags, confidence intervals, domain restrictions, policy guards
Lost if removed: The system transfers claims into contexts where they do not hold, corrupting cumulative knowledge.
Derivation: Any reliable cumulative learner must encode, infer, or test where learned structure applies.
