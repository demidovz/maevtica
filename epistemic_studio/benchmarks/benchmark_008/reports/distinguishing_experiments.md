# Distinguishing Experiments

## e01_cross_substrate_failure_prediction
- Target claims: c06_update_stability, c07_representation_theorem
- Design: Build a blinded benchmark of learning systems, institutions, markets, and evolutionary simulations. Code only the six abstract conditions, then predict which systems accumulate knowledge over long horizons. Compare against PAC-only, RL-only, Bayesian-only, and organizational-learning-only feature sets.
- Existing alternatives: PAC learning, reinforcement learning, Bayesian epistemology, organizational learning
- Decision rule: Novelty survives only if the six-condition model predicts cross-substrate failures significantly better than each domain framework without adding domain-specific features.
- Expected novelty value: 0.88

## e02_preserved_failure_ablation
- Target claims: c08_preserved_failures
- Design: Run matched research teams or LLM research collectives on multi-stage tasks with and without structured preserved failures. Hold total notes, time, and participants constant.
- Existing alternatives: Popperian criticism, ordinary lab notes, knowledge management
- Decision rule: The claim gains novelty if structured failure artifacts reduce duplicate hypotheses and time-to-falsification beyond ordinary notes and post-hoc summaries.
- Expected novelty value: 0.74

## e04_compression_predicts_future_reuse
- Target claims: c05_nonzero_cost, c09_question_refinement_compression
- Design: Track research projects over time and measure whether question-refinement compression predicts later reuse, fewer duplicates, and faster falsification after controlling for publication count, citation count, and model accuracy.
- Existing alternatives: MDL, bibliometrics, organizational learning
- Decision rule: Novelty survives if compression-through-question-refinement predicts future efficiency where existing complexity or impact measures do not.
- Expected novelty value: 0.71

## e03_scope_annotation_transfer
- Target claims: c04_scope_control
- Design: Compare model repositories, scientific claims, or organizational playbooks with explicit scope annotations against equivalent repositories without scope metadata across out-of-domain transfer tasks.
- Existing alternatives: statistical learning theory, external validity methods, RL generalization
- Decision rule: The Studio formulation adds value if scope metadata predicts transfer failure earlier than standard validation metrics alone.
- Expected novelty value: 0.69
