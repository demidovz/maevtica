# Working With User

## Язык и стиль

- Всегда отвечать по-русски.
- Пользователь предпочитает жёсткие формулировки без воды.
- Не делать длинных общих объяснений, если можно дать прямой вывод.
- Главное для пользователя: либо чёткое утверждение, либо следующая гипотеза на тест.

## Формат результата после теста

После каждого нового теста нужно давать четыре вещи:

1. Технический вывод.
2. Перевод "по-детски".
3. Следующую гипотезу.
4. Детский перевод следующей гипотезы.

Это стало явным требованием в ходе этой сессии.

Дополнение:

- в переводе "по-детски" не использовать слово `агент`;
- использовать простые слова вроде `штука`, `система`, `машина`, если без них не обойтись.

## Что пользователь любит

- Когда вместо разговоров запускаются реальные Python-скрипты.
- Когда каждая гипотеза раскладывается на тест.
- Когда результат формулируется как:
  - "`это проходит`",
  - "`это не проходит`",
  - "`сильная версия не проходит, проходит более точная версия`".
- Когда отрицательные результаты признаются прямо, без сглаживания.

## Что пользователь не любит

- Графики как главный результат.
- Размытые рассуждения без запуска кода.
- Когда ответ уходит в философствование вместо следующего шага.

## Автономный режим работы

Согласованный удобный режим:

- оптимально делать `4` итерации подряд без подтверждения пользователя;
- нормальный максимум `5`;
- после пакета итераций нужно возвращаться с коротким отчётом:
  - что подтвердилось,
  - что не подтвердилось,
  - как изменился тезис,
  - какой следующий узкий шаг лучше.

## Содержательные предпочтения пользователя

- Пользователь хочет строить именно исследовательскую линию через тесты.
- Интерес не в "сознании" как слове, а в том, куда ведёт тезис при последовательной проверке.
- Пользователь ценит детский перевод как способ проверить ясность формулировки.
- Пользователю важно, чтобы новый агент на другой машине мог быстро войти в проект без долгого reread чата.

## Практическое правило для новой сессии

Если новая сессия стартует без контекста, лучше начать так:

1. Прочитать [docs/SESSION_HANDOFF.md](C:\Users\user\workspace\maevtica\docs\SESSION_HANDOFF.md).
2. Прочитать [docs/PROJECT_HISTORY.md](C:\Users\user\workspace\maevtica\docs\PROJECT_HISTORY.md).
3. Спросить пользователя только о выборе следующей ветки, если это реально неочевидно.
4. Иначе сразу запускать следующий тест.
